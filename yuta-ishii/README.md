# テバシューター Phase 0

名古屋めし弾幕STGの1時間プロトタイプ。弾幕を避ける手触りの検証だけが目的。
仕様は [docs/spec.md](docs/spec.md) を参照。

## 起動

```
npm install
npm run dev
```

## 操作方法

| キー | 動作 |
| --- | --- |
| 矢印 / WASD | 8方向移動 |
| Shift | 低速移動(40%)+ 判定円の白色表示 |
| R | 即リスタート(被弾回数リセット) |
| T | 負荷テストモード切替(同時約800〜900発) |

被弾するとダメージはなく、画面フラッシュ + 全弾消し + 1秒無敵(点滅)で、左上の被弾回数が増えるだけ。
弾幕は8秒ごとに 🍗てばさき三連(aimed.json)→ 🍖みそかつリング(radial.json)→ 🍤えびふりゃー乱舞(scatter.json)の順でローテーションする。

## 見た目

画像ファイルは使わず、絵文字を実行時にCanvasへ描いてテクスチャ化している。
自機は🐓(名古屋コーチン)、ボスは🏯(名古屋城)、弾はパターンごとの名古屋めし絵文字。
絵文字はOS付属のフォントで描画されるため、macOS以外では見た目が多少変わる。

## 弾幕の難易度調整

`src/danmaku/patterns/*.json` の数値を変えるだけでよい(コード変更不要、devサーバー起動中なら保存で即反映)。

| フィールド | 意味 |
| --- | --- |
| `intervalMs` | 発射間隔(ms)。小さいほど激しい |
| `speed` | 弾速(px/秒) |
| `speedMax` | ランダム弾速の上限。指定すると speed〜speedMax でばらける |
| `count` | 1回に撃つ弾数 |
| `spreadDeg` | 弾の広がり(度)。360でリング、それ未満は扇 |
| `baseAngleDeg` | 基準角(90=真下)。aim=aimed では無視 |
| `aim` | `aimed`(自機狙い)/ `fixed`(固定角)/ `random`(範囲内ランダム) |
| `rotatePerShotDeg` | 発射ごとに発射角を回す量(度) |
| `bulletEmoji` | 弾の見た目の絵文字。省略時は🍗 |

調整例:

1. 自機狙いを激しく → `aimed.json` の `intervalMs` を 200→150、`count` を 3→5

2. リングを密に → `radial.json` の `count` を 24→36、`intervalMs` を 800→600

3. ばら撒きを速く → `scatter.json` の `speedMax` を 320→450

パターンの切替間隔(8秒)や自機速度などは `src/config.ts` にまとまっている。

## 負荷確認

Tキーで `stress.json`(全方位48発リング × 250ms間隔)に切り替わり、右上の FpsMonitor で
fps とアクティブ弾数を確認できる。ヘッドレスChromeでの計測では同時897発で fps 119〜120 を維持。
