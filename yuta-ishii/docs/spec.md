# テバシューター(名古屋めし弾幕STG)

名古屋めしテーマの弾幕シューティング。詳細仕様は docs/spec.md を必ず参照すること。

## 現在のフェーズ

**Phase 1: コアループ実装済み**
- Phase 0(弾幕回避プロトタイプ)の実装は完了。合格判定(開発者本人が5分遊んで「避けるのが気持ちいい」)は未実施
- Phase 1 で追加済み: ショット(🥚)+ボスHP、スコア、残機、被弾/撃破演出(パーティクル・シェイク)
- 見た目は絵文字テクスチャ(実行時Canvas生成、画像ファイル不使用)

## 技術スタック

- TypeScript(strict)+ Phaser 3 + Vite
- 描画は Phaser Graphics のみ(画像ファイル禁止 in Phase 0)
- 物理は Arcade Physics、当たり判定は circle body のみ使用

## 絶対に守る技術ルール

1. **弾は new しない**: 全弾 BulletPool 経由で取得・返却。Phaser.GameObjects.Group の get/killAndHide を使う
2. **当たり判定は自機グラフィックの約35%の円**。Shift 低速中は判定円を白く可視化
3. **移動・弾道は必ず delta(ms)ベース**。フレームレート依存の実装は不可
4. **弾幕パターンはコードに直書きしない**: src/danmaku/patterns/*.json に定義し PatternRunner が解釈する
5. 同時 800 発で 60fps を維持できない実装は作り直し。FpsMonitor を常時表示する

## 操作仕様

- 矢印 / WASD: 8方向移動
- Z / Space: ショット(押しっぱなしで連射)
- Shift: 低速移動(速度40%)+ 判定可視化
- 被弾: 画面フラッシュ + シェイク + ボスの弾全消し + 1秒無敵 + 残機-1(残機0で被弾するとゲームオーバー)
- ボスHPを削り切るとクリア(完食)
- R: 即リスタート
- T: 負荷テストモード切替

## コーディング規約

- npm 使用、ESM
- クラスは1ファイル1クラス、named export
- マジックナンバーは config.ts か pattern JSON へ
- コメントは「なぜ」だけ書く(何をしているかはコードで表現)
