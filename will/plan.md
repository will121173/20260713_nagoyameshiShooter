# 名古屋めしシューティング — 企画まとめ

1時間・ローカル完結のネタゲー。味噌カツ弾でよそもんを撃ち落とす。

---

## 1. アイデア

### 一言で

ブラウザだけで動くネタゲー「名古屋めしシューティング」— 味噌カツ弾で敵を撃ち落とせ。

### 背景・課題

- ハッカソンで「AI感」より**笑えてすぐ遊べる**デモが欲しい
- 1時間・ローカル完結で、セットアップや API キーに時間を取られたくない
- 名古屋ネタは説明なしでも伝わり、発表で空気が温まりやすい

### ターゲット

- ハッカソン審査員・観客（デモ視聴者）
- その場でキーボード操作して楽しめる人

### プレイ概要

- 画面下部の自機を左右移動し、上から降ってくる敵に名古屋めし弾を撃つ
- 弾＝味噌カツなど（絵文字 / 図形）
- 敵＝名古屋じゃない食べ物・ニセ名古屋めし
- スコア・制限時間・ゲームオーバーで一本のプレイループを閉じる

### ウリ

- ネタが一目でわかる
- 依存ゼロに近い（ブラウザを開けば即デモ）
- オフライン可

- [x] 当たり判定・スコア・時間切れ/ミスでゲーム終了できる

|---|------|------|

| 3 | 敵が上から出現し下へ移動 | [x] |
| 4 | 弾×敵の当たり判定と撃破・スコア加算 | [x] |
| 5 | 制限時間 60 秒（またはライフ制） | [x] |
| 6 | ゲームオーバー画面（スコア + リトライ） | [x] |

### Should（余裕があれば）

- [ ] 弾種切替（味噌カツ / 手羽先 / きしめん）
- [x] 簡易 SE（Web Audio ビープ）＋ネタボイス（SpeechSynthesis）

### 非機能

| 項目 | 内容 |
|------|------|
| 時間 | 実装 約 1 時間 |
| 環境 | ローカルのみ（オフライン可） |
| アセット | 絵文字 / Canvas。外部 API 不要 |
| 操作 | PC キーボード |
| ブラウザ | 最新 Chrome / Edge |

### 成功基準

1. 発表中に 30 秒〜1 分プレイし、スコアが出る
2. 「名古屋めし」ネタが一瞬で伝わる
3. 起動が `cd app && npm run dev` → ブラウザの 2 ステップ以内

### ゲームルール（MVP 固定）

| 項目 | 値 |
|------|-----|
| 制限時間 | 60 秒 |
| 操作 | ←→ 移動 / Space 発射 |
| 撃破 | +100 点 |
| 敵が下到達 | −1 ライフ（初期 3）※時間制だけでも可 |
| 弾 | 味噌カツ固定（Should で種類追加） |

## 3. 技術・構成

### スタック

| レイヤ | 選定 | メモ |
|--------|------|------|
| フロント | Vite + TypeScript + Canvas 2D | 最短のゲームループ |
| UI | 最小 HTML + CSS | タイトル/HUD は DOM |
| バックエンド | なし | 静的のみ |
| ストレージ | なし（任意で localStorage） | ハイスコアは Should |
| AI / API | なし | ローカル優先 |
| アセット | 絵文字 / 図形 | 著作権・収集ゼロ |

### なぜこれか

- React/Phaser は 1 時間には重い
- `npm run dev` だけで完結、API キー不要
- 絵文字で「味噌カツ弾」が伝わる

### 捨てた案

| 案 | 理由 |
|----|------|
| 単一 HTML | 動くが Vite の方が分割編集しやすい |
| Phaser / Pixi | 過剰 |
| React 中心 | 毎フレーム更新と相性が悪い |
| Pygame | デモ共有がブラウザより面倒 |

### 構成

```
[ブラウザ]
   └─ index.html
         ├─ DOM: タイトル / スコア / リトライ
         └─ <canvas>
               └─ requestAnimationFrame
                     ├─ input → update → render
```

### ディレクトリ（app 内）

```
app/
├── index.html
├── package.json
├── vite.config.ts
├── src/
│   ├── main.ts          … 起動・画面切替
│   ├── game.ts          … ゲームループ
│   ├── entities.ts      … 自機・弾・敵
│   ├── input.ts         … キー入力
│   ├── constants.ts     … スコア・時間・名称・絵文字
│   └── style.css
└── public/              … 空で可
```

### データフロー

1. Start → `playing`
2. 毎フレーム: 入力 → 弾/敵更新 → 衝突 → スコア/時間 → 描画
3. 時間切れ or ライフ 0 → `gameover` → リトライでリセット

### リスクと対策

| リスク | 対策 |
|--------|------|
| 時間不足 | Must のみ。見た目は絵文字 |
| 当たり判定 | AABB（矩形）のみ |
| 難易度調整 | `constants.ts` の定数 1 箇所 |
| 起動失敗 | 事前 `npm install`。最悪 `vite preview` |
