# 名古屋メシ シューティング

## プロジェクト概要
ブラウザで動く縦スクロール型シューティングゲーム「名古屋メシ シューティング」を作る。
詳細仕様は `src/要件.md` を正とする(内容が変わったらこのファイルより要件.mdを優先すること)。

要点:
- 単一HTMLファイル(HTML+CSS+JSを1ファイルに完全同梱)。外部ライブラリ・外部通信・画像ファイルは一切使用しない
- 描画はCanvas APIのみ。キャラクターはすべて絵文字を`fillText`で描画
- `requestAnimationFrame`で60fps。Canvasは480×640、画面中央配置
- 自機😋(残機3・無敵時間・自動連射)、雑魚3種(手羽先🍗/エビフライ🍤/台湾ラーメン🍜)、ボス(金のシャチホコ🐟)
- スコアは「摂取カロリー(kcal)」表示。localStorage (`nagoya_meshi_ranking`) にTOP10ランキングを保存
- タイトル→ゲーム→リザルト(イニシャル入力→ランキング)の画面遷移

出力物は `src/` 配下に単一HTMLファイルとして置く(例: `src/index.html`)。ビルドツール・パッケージ管理は導入しない。

## エージェント運用方針(司令塔 / 実装役)

このプロジェクトでは役割分担を明確にする。

- **司令塔(オーケストレーター)= このメインセッション**
  - 使用モデル: **Fable**(このセッションを開始する際は `/model fable` を推奨)
  - 役割: 要件.mdの解釈とタスク分解、実装方針の決定、実装エージェントへの指示、結果のレビューと統合、動作確認(ブラウザで実際に動かす)
  - 司令塔自身はまとまった実装コードを直接書かない。実装は必ず実装役に委譲する

- **実装役 = Agent tool 経由のサブエージェント**
  - サブエージェントタイプ: `implementer`(`.claude/agents/implementer.md` に定義)
  - デフォルトモデル: **Sonnet**(通常の実装タスクはこれで十分)
  - 難易度が高いタスク(当たり判定と衝突判定の調整、ボスの弾幕パターン制御、Web Audio APIでの効果音合成、状態リセット/メモリリーク対策など)は、Agent tool呼び出し時に明示的に `model: "opus"` を指定してエスカレーションする
  - 1つの実装タスク = 1エージェント呼び出しを基本とし、機能単位(自機/雑魚/ボス/UI遷移/ランキング/オーディオ等)に分割して委譲する

- 複数の実装タスクを並行で進めたい場合や、大規模な見直しをしたい場合のみ Workflow tool を使う(ユーザーが明示的に依頼した場合のみ)。通常の開発は司令塔からのAgent tool呼び出しで十分

## 動作確認
- 単一HTMLファイルなのでビルド不要。ブラウザで直接開いて確認する
- 実装が一区切りついたら、司令塔が実際にプレイして以下を確認する
  - 移動(矢印キー/WASD)、自動連射、当たり判定、無敵時間の点滅
  - 雑魚3種の出現・攻撃パターン、ボス登場条件と2種の攻撃パターン
  - カロリー加算、残機表示、ゲームオーバー/ボス撃破後のリザルト→ランキング→タイトルへの遷移とリセット
  - 長時間プレイでも画面外オブジェクトが確実に破棄され重くならないこと
