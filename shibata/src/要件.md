ブラウザで動く縦スクロール型シューティングゲーム「名古屋メシ シューティング」を作ってください。

# 絶対条件
- 単一HTMLファイル(HTML+CSS+JSをすべて1ファイルに含める)
- 外部ライブラリ・外部通信・画像ファイルは一切使わない
- 描画はCanvas API。キャラクターはすべて絵文字をfillTextで描画する
- requestAnimationFrameで60fps動作
- Canvasサイズは幅480×高さ640、画面中央に配置。背景は濃紺のグラデーション

# コンセプト
プレイヤーは腹ペコの挑戦者。上から襲ってくる名古屋メシを「撃つ=食べる」。
スコアは点数ではなく「摂取カロリー(kcal)」として表示する。

# 自機
- 絵文字😋、サイズ約40px、画面下部からスタート
- 移動:矢印キーとWASDの両対応。上下左右に移動可(画面外に出ない)。移動速度は毎秒300px程度
- 弾:自動連射(キー入力不要)。毎秒5発、白い米粒🍚(小さめ20px)が上方向へ毎秒500pxで飛ぶ
- 残機3。敵本体または敵弾に接触で残機-1、被弾後2秒間は無敵(点滅表示)
- 当たり判定は見た目より少し小さくする(半径の70%程度)

# 敵(すべて上から出現、倒すと「完食!」の小さなポップアップとカロリー加算)
1. 手羽先🍗(HP1、200kcal):ゆっくり下降しながらサインカーブで左右に揺れる。2秒ごとに骨(白い小弾)を自機狙いで1発撃つ。出現頻度:高
2. エビフライ🍤(HP1、350kcal):画面上の左右どちらかから斜めに高速で突進してくる。弾は撃たない。出現頻度:中
3. 台湾ラーメン🍜(HP3、600kcal):画面上部で静止し、3秒ごとに赤い弾を8方向に放射する。出現頻度:低
- ゲーム開始から時間経過で出現間隔が少しずつ短くなる(難易度上昇)

# ボス:金のシャチホコ
- 開始60秒後、雑魚の出現を止めてボス🐟(サイズ80px、金色に見えるよう背後に金色の円やグローを描画)が登場
- HP30。画面上部を左右に往復移動
- 攻撃パターン2種を交互に:(A)自機狙いの3way弾 (B)横一列の弾をランダムな1箇所だけ穴を空けて降らせる
- 撃破で3000kcal加算し「完食!!ごちそうさまでした!」表示→リザルトへ

# 画面遷移
1. タイトル画面:タイトル「名古屋メシ シューティング」、操作説明(移動:矢印キー/WASD、弾は自動発射)、ランキングTOP10、「Press Enter to Start」
2. ゲーム画面:上部に「摂取カロリー」と残機(😋の数)を常時表示
3. リザルト画面(ゲームオーバーまたはボス撃破後):最終カロリー表示→3文字のイニシャル入力(A-Z、矢印キーで文字選択+Enterで確定のアーケード風)→ランキング表示→Enterでタイトルへ

# ランキング
- localStorageにTOP10を保存(キー名: nagoya_meshi_ranking)
- 保存内容:イニシャル3文字、カロリー、日時
- カロリー降順で表示

# 演出(軽くてよい)
- 敵撃破時:絵文字が一瞬拡大して消える+「完食!」テキスト
- 被弾時:画面を一瞬赤くフラッシュ
- Web Audio APIで簡単な効果音(発射音、撃破音、被弾音)をコードで合成。音が出せない環境でもエラーにならないようにする

# 品質要件
- ゲームオーバー/クリア後にEnterで最初から遊び直せる(状態が完全にリセットされること)
- オブジェクトは画面外に出たら必ず配列から削除し、長時間プレイでも重くならないこと
